    APK重打包检测工具DroidMOSS

    2017-10-05
    Android, Papers

    DroidMOSS Android apk重打包检测工具

    功能：

    给定一个apk计算出该apk与官方市场apk的相似程度
    用来检测是否是重打包的apk

    步骤：
    1. 特征提取     
2. 生成指纹     
3. 基于指纹匹配

    步骤1：特征提取：

    1提取指令序列：
    • 需要足够robust，可以抵抗重打包
    • 为了对抗混淆：只获取操作码不获取操作数
    • 使用白名单过滤广告库的干扰
      2作者信息
      META-INF 目录包含了作者的证书， 名字联系方式等

    步骤2：生成指纹（将特征缩减，方便大规模apk检验）

    编辑距离？
    指令序列数量过多
    Fuzzy Hashing:
    将指令序列分为多块：重打包只能影响部分块
计算每块的hash
使用rolling hash 匹配块的边界（reset point）

    步骤3：基于指纹进行匹配

    将目标APK和已经搜集到的APK指纹进行匹配，用相似读判定重打包
计算两个指纹之间的编辑距离

    调研结果显示重打包的apk主要三种：

    1.  替换广告
2.  插入广告
3.  更改广告收入
4.  少量的apk植入后门

    缺陷：

    只考虑java指令没有考虑原生代码
